✅ Validate Event Pass

🔐 Pass Validation Security

This validation process includes:

  • Authorization Check: Verifies you have permission to validate passes
  • Digital Signature Verification: Ensures pass authenticity using SHA256+RSA
  • One-Time Use: Prevents pass reuse
  • Access Control: Only organizers and admins can validate

⚠️ Validation Requirements

You must have one of the following to validate passes:

  • Admin role (full access)
  • Organizer role for the specific event
  • Explicit validation permission granted via ACL

Validation Process Flow

  1. Pass Code Lookup: Find the pass in database
  2. Authorization Check: Verify validator permissions
  3. Signature Verification: Validate digital signature
  4. Usage Check: Ensure pass hasn't been used
  5. Mark as Used: Update pass status and timestamp
